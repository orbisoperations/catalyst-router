B: Adapter Cards Grid

Replace the adapter list with cards showing health, protocol, origin node, response time, and which peers have the route. More info at a glance, follows the ServiceCard pattern from the Services tab.

Backend approach (same for both)

  • Orchestrator periodically checks /health on local adapters (configurable interval, 30s default)
  • Health status (up/down/unknown, responseTimeMs, lastChecked) added to route metadata
  • Propagated to peers via iBGP alongside existing route data
  • No /health endpoint → show "unknown" (not "down")
  • /api/state includes health data — web-ui just renders it
